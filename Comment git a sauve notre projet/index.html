Il était une fois...

Un éditeur de progiciel dont le produit avait plus de 10 ans
Il vivait heureux avec toutes ses versions et tous ses clients dans son palais en compagnie de son fidèle conseiller SVN
Mais ses fidèles sujets commençaient à gronder...
Car chaque client avait une version différente du logiciel et les petits elfes commençaient à peiner lors des mises à jour
le fidèle conseiller SVN affaiblissait le royaume en consommant de plus en plus de ressources
les artisants pleuraient à chaque report d'une branche à l'autre
le roi n'avait aucune visibilité sur la dernière version stable du travail réalisé par les artisants
Alors...
Les artisants décidèrent que c'en était trop !
Ils proposèrent plusieurs solutions pour remédier à ses problèmes
  • Faire des versions plus courtes
  • Obliger les clients à être tous dans la même version
  • Faire du ménage pour que SVN arrête de grossir
  • ...
Quand soudain, on entendit les mots suivants ...
"Et si on changeait de système de gestion de versions ?"
Euh... OK, mais pour prendre quoi ?
  • CVS
  • Mercurial
  • Git
  • ClearCase
  • ...
"J'ai entendu parler de Git et je veux bien creuser le sujet pour voir si ça peut nous aider !"
Le roi accepta de laisser les artisants étudier la question
Un beau jour, les artisants arrivèrent pour présenter les conclusions suivantes :
  • Problème espace disque : OK
  • Fusion de branches : OK
  • Version stable à un instant t : OK en changeant notre façon de travailler
Il présenta une nouvelle façon de travailler : GitFlow
GitFlow (schéma)
Tous les sujets du royaume semblaient partants et se mirent d'accord pour commencer (oui, car c'était un royaume auto-organisé).
Mais... par où commencer ?
Ils choisirent un projet qui servirait de cobaye
"Oh celui là est petit ! Et en plus, il est fortement lié à SVN et devra évoluer de toute façon !"
Le projet cobaye fut mis à rude épreuve, mais la conclusion de cette migration fut :
"C'est parti !"
Il fallut adapter les outils de travail des artisants pour qu'ils s'adaptent à cette nouvelle façon de travailler (Eclipse, Tortoise, Ligne de commande, ...)
Il fallut également former tous les compagnons et les accompagner dans leurs premiers pas sur le champ de bataille
Les sujets attendirent le bon moment pour destituer le fidèle SVN (la fin d'une version).
Quand tous furent prêts, le script final fut exécuté.
Le fidèle SVN fut néanmoins gardé.
Les anciennes versions restèrent accrochées à lui (avec reports sous Git).
A terme, il pourrai enfin reposer en paix...
Depuis cette rude bataille, le royaume vit paisiblement au rythme des "git push", gut commit, "git flow release start", ...
Quelle sera leur prochaine aventure ?
